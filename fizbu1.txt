1. Product Vision & Scope
Fizbo is Portugal's first seller preparation platform. It solves the single biggest reason
private property sales fail: the seller arrives unprepared — missing documents, no credible
price, and no qualified buyer pool.
Fizbo does not replace agents. It replaces the chaos that precedes a sale. A Fizbo-prepared
seller has complete documentation, a data-backed price, a professional listing, and access to a
verified off-market buyer pool — before they speak to any agent or post any public listing.
Mission: Make every Portuguese property seller as prepared as a professional agent —
through technology, data, and guided processes that eliminate information asymmetry and
reduce transaction failure.

2.2 The Operator (Internal User)
The Fizbo operator manages all document procurement tasks that cannot be automated. They
need a queue-based dashboard.
• Reviews incoming orders and confirms document requirements
• Contacts municipal offices, banks, condominium administrators on behalf of seller
• Coordinates perito qualificado appointments for energy certification
• Updates document statuses (triggers automated seller notifications)
• Handles exceptions: blocked orders, missing heirs, unresponsive municipalities

3.1 DocCheck Tool — Input Questions
8 questions, branching logic, completing in under 90 seconds. Email required to view full result.
1. What type of property? (Apartamento / Moradia / Terreno / Outro)
2. Is the property in a condominium? (Yes / No / Unsure)
3. When was the building constructed? (Pre-1951 / 1951–1990 / 1991–2007 / Post-2007)
4. Do you have a mortgage on the property? (Yes / No)

5. How did you acquire it? (Purchase / Inheritance / Divorce settlement / Gift / Other)
6. Is this your primary residence? (Yes / No — rented / No — vacant)
7. Do you have a valid energy certificate under 10 years old? (Yes / No / Don't know)
8. Are all owners available to sign? (Yes / One abroad / One deceased / Disputed)

4. User Flows
4.1 Standard Seller Flow (P1)
1. Arrives on fizbo.pt (organic / referral / PropCheck CTA)

3. Runs DocCheck — 8 questions, unauthenticated

4. Sees: document gap report + recommended tier + price
5. Clicks "Get my documents sorted" → tier selection → Stripe Checkout
6. Stripe success → account created automatically → property onboarding

 Inngest DocReady workflow triggered
8. Document collection phase (15–20 working days)

 Notifications: Email + WhatsApp on each status change

9. "Your documents are ready" notification sent
 All documents downloadable from dashboard
10. SmartCMA CTA displayed → price report generated on click
11. "Activate your off-market listing on Reserva" CTA
4.2 Non-Resident Additions (P2)
• After DocCheck result: "Are you resident in Portugal?" → if No: fiscal rep options shown
• Procuração requirement explained with template download link
• Mais-valias calculation included and shown at onboarding
• All comms available in English (language set at account creation)
• Casa Pronta videoconference option surfaced for remote escritura
4.3 Inherited Property Flow (P3)
• DocCheck identifies via Q5 (Inheritance) + Q8 (not all owners available)
• Habilitação de Herdeiros is shown as step zero — prerequisite for all other docs
• If heir abroad: Apostille requirement explained with timeline (2–4 weeks)
• Timeline gating: other documents marked "Waiting on Habilitação" until step zero
complete
• 10% stamp tax shown for non-exempt heirs (siblings, friends, etc.)

5.2 Pricing & Payment Rules
• All prices are VAT-exclusive. VAT breakdown shown at Stripe Checkout.
• If seller confirms valid energy certificate (Q7 = Yes): subtract €100 from displayed price
automatically — shown as line item at checkout.
• Refund policy: 100% if no documents requested within 5 working days. 50% if work
started but no docs obtained. Zero once any document is obtained.
• DocExpress requires payment confirmation before work begins.
• Stripe handles all card processing. Zero card data stored on Fizbo servers.

5.4 Acceptance Criteria
DocCheck
• AC1: DocCheck runs unauthenticated. Email required only to view full result.
• AC2: Maximum 8 questions. Result page loads in < 2 seconds.
• AC3: Result shows: document checklist, recommended tier + price, timeline estimate,
CTA.
• AC4: Result persists for 24 hours in the same browser session.
• AC5: Email capture creates a lead record even without purchase.
DocReady Flow
• AC1: Stripe payment → account creation → property onboarding completes in < 3
minutes on mobile.
• AC2: Caderneta upload → OCR result in < 30 seconds. If OCR fails: manual entry form
shown automatically.
• AC3: Document statuses update in real-time (Supabase Realtime). No manual refresh.
• AC4: WhatsApp notification sent on every status change where phone number is on file.
• AC5: Operator dashboard shows all active orders without page refresh (Realtime).
• AC6: SmartCMA PDF generated in < 60 seconds of order completion trigger.

6. Non-Functional Requirements
6.1 Performance
• Public pages (DocCheck, landing, pricing): < 2.5s LCP on 4G mobile
• Seller dashboard: < 1s to first meaningful paint
• OCR processing: < 30 seconds end-to-end
• SmartCMA generation: < 60 seconds end-to-end
6.2 Security & GDPR
• NIF encrypted at application level (pg_crypto) — never stored in plaintext
• Document files in private Supabase Storage — no public URLs, signed URLs expire in 1
hour
• Right-to-deletion endpoint: removes all personal_data records + Storage files
• Data retention: 5 years for order records (fiscal obligation). Contact data deleted 90 days
post-completion if no further activity.
• All data stored in EU (Supabase EU region)
6.3 Localisation
• Primary language: PT-PT. All seller-facing content written in PT-PT.
• English: available via /en URL prefix. Separate URL (not toggle) — required for SEO
hreflang.
• Mobile-first: entire seller journey must complete on mobile without friction.
